# **Задача 3: Архитектура инфраструктуры для High-Volume Outreach**

Вводные: Обслуживание 1200 email-адресов (seats).  
Цель: Минимальный OPEX, высокая отказоустойчивость, Deliverability \> 95%.

## **1\. Концепция: Гибридная модель "Агрегатор \+ Tiered Sender"**

Прямая покупка 1200 лицензий Google Workspace/Office 365 экономически нецелесообразна (\~$7200/мес). Поднятие собственного SMTP (Postfix) на "голом" железе несет критические риски попадания IP в Blacklist.

Предлагаемое решение:  
Использовать гибридный подход, разделяя ящики на уровни важности:

* **Tier 1 (High Priority \- 20%):** Google Workspace / Microsoft 365\. Используются для VIP-клиентов и этапов "дожима". Самая высокая репутация домена.  
* **Tier 2 (Bulk Volume \- 80%):** Специализированные провайдеры инфраструктуры для аутрича (Mailforge, Maildoso) или кастомные сборки. Они дешевле в 3-4 раза.  
* **Оркестратор:** Единая платформа управления (Smartlead или Instantly.ai).

## **2\. Технический стек**
| Компонент      | Решение                  | Обоснование                                                        |
|----------------|--------------------------|--------------------------------------------------------------------|
| DNS & Domains  | Cloudflare (Free Tier)   | ыстрое управление DNS через API. Лучшая защита от DDoS.            |
| Sending Tool   | Smartlead.ai / Instantly | "Умная" ротация ящиков, Unified Inbox, авто-прогрев (Warm-up).     |
| Infra Provider | G-Suite + Mailforge      | Баланс цены и качества доставки.                                   |
| Automation     | Python / n8n             | Мониторинг здоровья доменов, алерты в Telegram при падении метрик. |

### **Настройки DNS (Critical)**

Для каждого домена настраивается "Тройка аутентификации" \+ Custom Tracking:

1. **SPF:** Строгое ограничение IP (v=spf1 include:... \-all).  
2. **DKIM:** 2048-bit подпись.  
3. **DMARC:** Политика p=quarantine на старте, переход к p=reject через 4 недели.  
4. **CNAME Tracking:** Отдельный поддомен (напр. tr.domain.com) для трекинга, чтобы не использовать общие домены сервиса рассылки.

## **3\. Ротация и Мониторинг**

### **Алгоритм ротации (Smartlead/Instantly logic):**

1. **Warm-up Pool:** Новые ящики греются 14-21 день (рост с 2 до 30 писем/день).  
2. **Active Pool:** Рабочие ящики. Лимит: 30-50 писем/день на ящик.  
3. **Cooldown:** Если Reply Rate падает ниже порога или домен ловит "Soft Bounce", он автоматически выводится в отлежку на 7 дней.

### **Мониторинг:**

* Автоматическая проверка доменов в блеклистах (Spamhaus, SORBS) через API.  
* Контроль **Sender Reputation** через Google Postmaster Tools.

## **4\. Риски и Mitigation Plan**

* **Риск:** "Burned Domain" (Домен попал в спам).  
  * *Решение:* Изоляция. Один клиент \= изолированный пул доменов. Если горят домены Клиента А, Клиент Б не страдает. Замена домена из резерва за 24 часа.  
* **Риск:** Блокировка провайдером за спам.  
  * *Решение:* Spintax (рандомизация текста) и AI-персонализация первой строки. Хеш каждого письма должен быть уникальным.

## **5\. Оценка стоимости (OPEX estimate)**

* **Домены (400 шт.):** При схеме "1 домен \= 3 ящика". \~$350/мес (амортизация).  
* **Инфраструктура:**  
  * 200 Premium (Workspace): \~$1200  
  * 1000 Standard (Bulk): \~$1500  
* **Софт (Sending Tool):** \~$300/мес (Unlimited seats).

**Итого:** **\~$3350/мес** (вместо $8000+ при классической схеме).
